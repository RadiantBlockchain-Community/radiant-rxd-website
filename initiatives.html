    Radiant Initiatives

    Initiatives and developments towards the growth, strength, awareness and adoption of the Radiant network and ecosystem around the world.

    • H1 2026
    • 2022 - 2025

    Radiant Core Node

    The goal of Radiant Core Node is to create sound money and a digital value transfer system that is usable by everyone in the world.

    Early 2026 (v2.0.0 / V2 Hard Fork): Activated at Block 410,000, this upgrade fundamentally expanded the network's computational limits. It introduced OP_BLAKE3 and OP_K12 opcodes, allowing developers to validate Proof-of-Work algorithms natively at the script level without relying on indexers. It also re-enabled crucial bitwise and math opcodes for complex smart contracts.

    March 2026 (v2.2.0): Deployed deep network hardening protocols. This release focused on peer-to-peer layer stability, optimizing synchronization, and ensuring uniform block propagation ahead of the April supply halving.

    June 2026 (v3.1.1 & v3.1.2): Rapid patch deployments adjusting node performance parameters and delivering immediate consensus stability fixes following the V2 hard fork integrations.

    Mid-2026 (Future Roadmap): Active conceptualization on future protocol drafts via the REP system, including potential P2P Encryption (BIP324) and OP_CHECKTEMPLATEVERIFY (CTV) integrations. The Radiant Core website is actively maintained for network representation.

    Radiant Core Node GitHub | Radiant Core Website

    Radiant Enhancement Proposals (REP)

    Mid-2026 (Process Formalization): Established the formal REP repository under the Radiant-Core organization as the primary mechanism for proposing major new features, collecting community input, and documenting protocol design decisions. This includes finalizing the DNS Seeder Infrastructure (REP-1) and drafting the comprehensive Glyph Protocol v2 standards (REP-3001 through 3004).

    REP GitHub

    Photonic Wallet

    A non-custodial RXD wallet for minting, transferring and swapping tokens (FT and NFT) and WAVE names on the Radiant blockchain.

    Early 2026 Standardized the derivation path to SLIP-0044 registered coin type 512 (m/44'/512'/0'/0/x) to align with Core Node v3.0.0 and the broader Electron Wallet ecosystem. Also natively integrated Glyph v2 dMint contract support.

    March 2026 Stabilized the build environment by resolving core errors and deploying the necessary VPS infrastructure to host and scale the non-custodial web app.

    Late May 2026: Executed a major UI/UX security overhaul. Hardened client-side cryptographic functions and introduced forced transaction confirmation modals across all outgoing screens to mitigate accidental or malicious transfers.

    Mid-June 2026: Transformed from a basic token wallet into a decentralized trading terminal. Deployed direct client-side integration for Constant Product Market Maker (CPMM) micro-pools, on-chain orderbook trading, and committee oracle markets.

    Ongoing (Mid-2026): Actively executing continuous, major overhauls of the wallet architecture - now maintained heavily under the Radiant-Core organization - to support the rapid expansion of complex ecosystem features, decentralized exchanges, and cross-chain capabilities.

    Photonic Wallet Web App | Photonic Wallet GitHub

    RXinDexer & ElectrumX

    H1 2026 (Database Architecture Overhaul): Transitioned the backend data layer to a production-optimized RocksDB setup. This architectural shift slashed steady-state RAM consumption by ~52% (dropping from ~1.17GB to ~561MB), massively reducing the capital expenditure required to host enterprise-grade network indexers.

    H1 2026 (Advanced Data Tracking): Expanded indexing capabilities to fully support real-time parsing for both the newly integrated WAVE naming system (domains/subdomains) and the on-chain Swap DEX order books.

    Mid-2026 (Glyph & Docker Integration): Upgraded the bespoke RXinDexer logic to natively support intricate Glyph v1 and v2 token indexing (including decoding CBOR metadata payloads). Shipped full-stack Docker deployments allowing node operators to instantly spin up synced UTXO/history indexers.

    RXinDexer GitHub | ElectrumX GitHub

    Glyph Miner

    A multi-algorithm cryptocurrency miner for tokens that follow the Glyphs Protocol, using layer 1 mining contracts.

    Early to Mid 2026: Rolled out multi-algorithm support (SHA256d, Blake3, KangarooTwelve, Argon2id-Light) to align with the new Layer-1 mining contracts introduced in the V2 Hard Fork. Shipped both a dedicated CLI miner for hardware optimization and a user-friendly browser extension utilizing WebGPU for casual mining.

    Mid-June 2026: Upgraded connectivity and API infrastructure. Migrated the default REST API endpoint to glyph-miner.com and introduced progressive connection logic to stabilize the handshake between high-efficiency mining hardware and network Stratum proxies.

    Ongoing (Mid-2026): Currently undergoing comprehensive architectural overhauls to continuously iterate on mining stability, implement multiple Dynamic Difficulty Adjustment (DAA) modes, and maximize algorithm efficiency for decentralized minting.

    Glyph Miner Web App | Glyph Miner GitHub

    WAVE Naming System & Protocol Standards

    A peer-to-peer naming service built natively on the Radiant blockchain.

    H1 2026 (REP-3011 Definition): Formalized the protocol for an on-chain, decentralized character-based prefix tree index. This system uses zone records to map human-readable names to standard cryptographic addresses and content hashes.

    Late May 2026: Unified protocol constants into the @radiantblockchain/constants repository (which feeds limits, opcodes, and flags to rxdeb, RadiantScript, and radiantjs), ensuring that all ecosystem applications parse parameters identically.

    Late May / Early June 2026 (ElectrumX v1.4.1): Deployed the specific RPC methods (wave.resolve and wave.check_available) allowing indexers to natively query the prefix tree and return active domain states to front-end applications.

    WAVE Name Explorer | WAVE REP-3011 | Radiant Constants GitHub

    Komodo DeFi Framework & DEX Integration

    Early 2026 (Native Chain Support & Sighash Implementation): Successfully added native Radiant (RXD) chain support to the Komodo DeFi Framework (KDF). Specifically, developed and merged the ForkIdRxd sighash signing path (via PR #2713), establishing the fundamental cryptographic bridge required for cross-chain compatibility.

    May 2026 (Gleec DEX Deployment): Because the Komodo DeFi Framework powers the backend for decentralized trading engines and multi-chain atomic swaps, this codebase integration directly enabled Radiant's rollout onto the Gleec DEX in May. This effectively allowed RXD to be securely traded peer-to-peer against other assets without relying on centralized exchange intermediaries.

    GRAVITY Protocol Implementation

    A peer-to-peer cross-chain exchange using SPV proofs and covenants on the Radiant blockchain.

    H1 2026 (Trustless PoW Swaps): Spearheaded the technical implementation surrounding the newly released Gravity Protocol whitepaper. This protocol acts as a multiway cross-chain operation, using Simple Payment Verification (SPV) proofs and native Radiant instructions to facilitate direct, peer-to-peer transactions between independent Proof-of-Work blockchains.

    H1 2026 (Counterparty Risk Elimination): By building out the Gravity architecture, the foundation was laid for zero-intermediary cross-chain trading. It leverages Radiant Photons as a form of trustless bond or collateral, fundamentally expanding how RXD natively interacts and swaps with the broader Proof-of-Work ecosystem.

    Gravity Prototype GitHub | Gravity Whitepaper

    Radiant Ledger Hardware App

    Sideloadable Ledger Nano S Plus app for signing Radiant (RXD) transactions.

    April 2026 (Hardware Wallet Orchestration): Led the planning, documentation, and codebase orchestration (with code living in Zyrtnin-org/app-radiant) to bring native Radiant support to Ledger hardware wallets.

    April 2026 (Cryptographic Preimage Fixes): Solved highly specific integration roadblocks regarding Radiant's UTXO signing mechanics. Specifically, engineered fixes for transaction preimage issues (such as resolving the hashOutputs hashing logic that was failing in standard implementations), ensuring Ledger devices can securely and correctly sign Radiant's custom smart contracts.

    Current Status (Mid-2026): Development is currently on hold pending a mandatory external audit required for official Ledger submission. The team is gathering quotes to cover the audit costs and exploring alternative funding options, which may include reallocating the indexer bounty to bypass this roadblock.

    Radiant Ledger App GitHub | Usage Guide | Planning docs

    Network Launch & Architecture

    Radiant is a UTXO-based L1 blockchain with native on-chain token indexing, Turing-complete smart contracts, and trustless asset validation - secured by SHA512/256d PoW.

    June 2022 (Genesis): The Radiant (RXD) network was officially launched on June 21, 2022. Initiated organically by a group of miners, the blockchain was bootstrapped with a pure Proof-of-Work (SHA512/256d) consensus, featuring zero initial funding, no ICO, and a 0% pre-mine. The architecture introduced a high-performance UTXO-based Layer 1 designed for extreme throughput (defaulting to 256 MB block sizes) and EVM-like Turing Complete programmable smart contracts.

    August 2022 (Whitepaper Publication): The foundational "Radiant Blockchain System Design" whitepaper was released. It detailed the core design of a zero-compromise peer-to-peer digital asset system, prioritizing induction proofs, parallel processing capabilities, and highly efficient network scaling to handle up to 1,000 tx/sec natively without secondary layers.

    2022 (Project Gravity Genesis): The conceptual foundations for the Gravity Protocol were established in late 2022. The initial proposal detailed a peer-to-peer protocol using SPV (Simple Payment Verification) inclusion proofs to execute decentralized cross-chain trades, laying the critical groundwork for the fully realized 2026 trustless implementation.

    Core Upgrades & Foundational Tooling

    August - September 2022 (Electron Wallet): The first desktop client was deployed, the classic Electron-Radiant wallet (a fork of Electron-Cash), allowing early adopters to securely hold and transfer assets.

    November 2022 (Zeus v1.2.0 Hard Fork): Activated at block 62,000, this upgrade solidified network terminology by officially shifting the base atomic unit from "Satoshis" to "Photons" (PHO), establishing the standard that 1 RXD = 100,000,000 Photons.

    2023 (Infrastructure Scaling): Critical backbone infrastructure was continually hardened, including the highly optimized Python-based ElectrumX indexer service (tested and proven to handle block sizes exceeding 4 GB) and the initial radjs library toolkit.

    April 2024 (Energy v1.3.0 Hard Fork): Deployed a major consensus upgrade introducing powerful new introspection capabilities. This included the OP_PUSH_TX_STATE opcode, allowing smart contracts to securely reflect on, access, and validate current transaction IDs, input amounts, and total outputs entirely on-chain.

    2024 - 2025 (MuonPay SDK): Developed by the community, the MuonPay SDK was authored as a streamlined integration kit for processing native Radiant (RXD) payments on independent websites. The SDK features backend infrastructure support, private key and seed phrase management, and an automated postback (webhook) URL system, allowing merchants and web developers to seamlessly accept peer-to-peer RXD transactions without relying on centralized payment gateways.
